SDBG

Eclipse JavaScript debugger with Source maps support

Follow me onGitHub

What is this?

JavaScript debugger for Eclipse, which supports debugging of Compile-to-JavaScript languages that generate Source Maps.

Currently, it has built-in support for GWT, but it should be easy to add support for any other compile-to-JavaScript language like TypeScript, CoffeeScript, Kotlin, etc.

It has been adapted from the Dart project, and modified for use as a generic JavaScript debugger.

How to install?

Eclipse 3.7 or later is required.

Install from the SDBG P2 repository by using the following sequence of Eclipse commands:
  • Help menu -> Install New Software... -> Add...
  • (The "Add Repository" dialog will appear)
  • For "Name", enter "SDBG" or something similar
  • For "Location", fill-in the following URL: http://sdbg.github.io/p2
Alternatively, you can download the latest SDBG P2 repository ZIP and install it by using the following sequence of Eclipse commands:
  • Help menu -> Install New Software... -> Add... -> Archive...

Status

The current implementation can be considered beta quality.

What is working:
  • Launching a new instance of Chrome for debugging
  • Connecting to a running Chrome instance for debugging
  • Debugging code running in the Chrome for Mobile browser on Android (including port forwarding a website running locally on your PC to Chrome for Mobile!)
  • Setting breakpoints in Java, including setting breakpoint in code coming from JAR libraries
  • Opening the Java (or other source language) source files instead of the obfuscated JavaScript
What is mostly working:
  • Deobfuscating the stacktrace. Works but does not look 100% like e.g. the JDT Debugger stacktrace. For example, the methods are not deobfuscated to their Java equivalents
Future work:
  • Prio A: Deobfuscating the Variable tabs. Currently, Eclipse will display the Variables tab with your JavaScript objects in them (not yet translated back to Java)
  • Prio A-B: Support for other source languages besides Java (e.g. CoffeeScript, TypeScript etc.); should require relatively little effort, as the interfaces for implementing such support are now well defined
  • Prio A-B: Support for Safari, Opera and Mobile Safari; should be easy as all these currently use the same Remote Debugging protocol as Chrome
  • Prio B: Support for Internet Explorer and Firefox

Testing

Once you are running the plugin, you can test it using the gwtproject.zip file found in the root of the repository:
  • Import the project into your Eclipse workspace by using the following commands: File -> Import... -> Existing Projects into Workspace -> Next -> Select Archive file
  • Next, open the "Debug Configurations" dialog and invoke the "gwtproject" launch configuration which will be available under "Chrome launch" in the launchers' tree
  • Put a breakpoint in the method called "toggleMenu", in the (only) Java file in the gwtproject project - GWTProjectEntryPoint.java
  • Click on "Articles" or "Documentation" in the "Table of contents" tree at the GWT Project Page
  • Observe the breakpoint being hit

Build Instructions

To clone the Git repository (forgive the large download; we have retained all Dart history for now): 
git clone git@github.com:sdbg/sdbg.git
To develop in Eclipse:
  • Import the root of the source as a Maven project
  • Inside the com.github.sdbg.debug.ui project, you will see a launch configuration called SDBG.launch
  • Running this configuration will open a new instance of Eclipse with the debugger plugin installed
To build everything from the command-line, including the P2 repository: 
mvn install
If you encounter any issues, please post to the SDBG Google Group
